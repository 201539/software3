# 当前项目状态摘要

更新时间：2026-04-21 17:40

## 项目定位
这是一个 AI Coding Agent Web MVP，目标是提供一个可让 Agent 直接读写工作区文件、执行命令、并在前端呈现对话、编辑器和文件树的开发环境。

## 当前界面结构
页面已经改成三栏同一行布局：
- 左侧：对话区
- 中间：编辑器
- 右侧：工作区文件树

三栏之间使用细线分隔，支持鼠标拖拽调整宽度。

### 布局特性
- 三栏可拖拽调整
- 有最小宽度限制，避免缩得太窄
- 拖拽宽度会持久化到 `localStorage`
- 小屏幕下会自动降级为单列布局

## 文件树状态
文件树已经做成更像 IDE 的形态：
- 支持文件夹展开 / 收起
- 支持展开状态持久化
- 支持右键菜单
- 支持自定义弹窗
- 支持新建文件 / 新建文件夹
- 支持重命名
- 支持删除
- 空工作区时不再显示空状态提示文案

### 目前的新建流程
点击“新建”后，会出现下拉菜单：
- 新建文件
- 新建文件夹

选择其中之一后，会直接弹出类似重命名的输入弹窗，不再经过中间选择页。

## 编辑器状态
编辑器已经支持：
- 打开文件
- 手动修改
- 自动保存

### 自动保存行为
- 编辑器内容变化后会自动写回后端
- 失焦时也会触发保存
- 保存成功后会同步工作区树

## Agent 执行结果展示
Agent 的工具调用结果仍然留在对话区，但已经改成可折叠形式：
- 默认只显示一行摘要
- 点击三角标后展开完整详情
- 工具调用结果不再直接铺满对话区

## 工作区启动状态
当前工作区已经改成**空工作区启动**：
- 不再预置 `src/App.tsx`
- 不再预置 `src/main.tsx`
- 不再预置 `package.json`
- 初始状态下文件树为空

### 根目录说明
- 逻辑上仍然存在工作区根目录
- 但 UI 上不显示一个单独的根节点
- 空工作区就是空树，不会显示示例目录

## 后端能力
当前后端已经支持：
- `PUT /api/file`：创建 / 更新文件
- `PUT /api/folder`：创建目录
- `POST /api/item/rename`：重命名文件或目录
- `POST /api/item/delete`：删除文件或目录
- `GET /api/workspace`：获取文件树
- `GET /api/file/:path`：读取文件
- `POST /api/agent/preview`：Agent 预览与工具调用流

## 文件系统管理
`workspace-manager` 已经支持：
- 空树初始化
- 文件写入
- 真目录创建
- 文件 / 文件夹重命名
- 文件 / 文件夹删除
- 输出最新 tree 供前端刷新

## Agent 编排策略
当前 Agent 编排已经开始向分层演进，核心是“Planning -> Execution -> Review -> Summary”的轻量阶段化流程：
- 先由 `context-builder` 生成本轮上下文
- `agent-core` 先记录 planning 阶段，构建 taskState
- 再把上下文交给模型请求，让模型决定是否调用工具
- 工具调用执行完成后进入 review 阶段
- 最后汇总输出 summary，并把 taskState 一起返回前端

### 当前编排特点
- 仍然不是完整的多步 Planner / Executor / Reviewer 独立服务架构
- 但已经不再是单纯“一次模型请求后直接回传”的扁平流程
- 运行时会记录阶段、工具调用、工具结果和摘要
- 工具结果会通过 SSE 流回传到前端，并以可折叠工具块展示
- 当前仍然更适合 MVP 验证，但已经具备分层编排的雏形

## 上下文控制策略
当前上下文控制是轻量启发式策略，核心逻辑在 `packages/context-builder`：
- 先读取整个工作区文件列表
- 再基于用户 prompt 和当前选中文件做相关性排序
- 只保留前几个最相关文件进入上下文
- 文件内容会做截断压缩，长内容只保留首尾关键部分
- 通过 `contextBudget` 控制文件数量和总字符数

### 当前上下文规则
- 最多包含 8 个文件
- 最大字符数约 12000
- `package.json`、前端源码文件会略微加权
- 当前选中文件会优先保留更多内容
- 超出预算后停止继续追加文件摘要

### 上下文输入包含
- `prompt`
- `selectedFile`
- `selectedFileContent`
- `workspaceSummary`
- `contextBudget`

## 交互细节
- 右键文件夹可新建、重命名、删除
- 右键文件可重命名、删除
- 新建按钮打开下拉菜单
- 新建文件 / 文件夹都使用自定义弹窗
- 删除使用危险确认弹窗
- 重命名使用自定义输入弹窗

## 当前已知注意点
- 页面主要是 MVP 级别，但核心交互链路已经闭环
- Agent 的工具结果会以折叠块形式显示在对话中
- 文件树为空时不显示任何提示内容，这是当前有意的设计

## MVP 方案对照表

下面是当前实现与 `@文档/MVP最小实现方案.md` 的对照结果。

| 能力项 | 文档要求 | 当前状态 | 说明 |
|---|---|---|---|
| Chat 输入与流式输出 | 必做 | 已完成 | 支持用户输入、SSE 流式返回、任务状态展示 |
| 工作区文件树 | 必做 | 已完成 | 支持展示、打开、刷新、展开/收起、右键菜单 |
| 代码编辑器 | 必做 | 已完成 | 支持查看、手动编辑、自动保存 |
| 项目骨架生成 | 必做 | 部分完成 | 目前是空工作区启动，已去掉默认模板；还未做一键模板生成 |
| Agent 修改代码 | 必做 | 已完成 | 可读文件、写文件、结果回显，但修改链路仍偏轻量 |
| 受控命令执行 | 必做 | 部分完成 | 已支持命令执行，但还缺风险分级、白名单和更完整审计 |
| 版本快照与回滚 | 必做 | 未完成 | 目前还没有真正的快照创建与回滚流程 |
| Preview Runner | 必做 | 未完成 | 还没有完整预览服务、URL 返回和构建错误收集 |
| State Store | 必做 | 未完成 | 还没有完整的项目 / 会话 / 任务 / 版本 / 日志持久化层 |
| Context Builder 压缩 | 必做 | 已完成 | 已做相关性排序、限量和截断压缩 |
| Planner / Executor / Reviewer / Summarizer | 架构目标 | 已完成 | 已拆分为独立模块，`agent-core` 负责串联阶段化流程，当前已具备轻量分层编排 |
| Web UI 三栏布局 | 不是文档原始条目，但属于当前 UI 改进 | 已完成 | 已做可拖拽三栏布局、折叠工具结果、空工作区启动 |
| 工具调用可折叠展示 | 不是文档原始条目，但属于当前交互优化 | 已完成 | 工具结果留在对话区但以折叠块呈现 |

### 当前结论
- 现在的实现已经验证了 MVP 的核心闭环，但还没覆盖文档里全部“最小实现方案”
- 最核心的缺口集中在：**模板生成、快照/回滚、Preview Runner、State Store、完整编排分层**
- 当前 Agent 编排更适合称为“轻量 MVP 编排”，不是文档设想的完整版本

## 后续可继续做的方向
- 给工具调用块做更细致的视觉优化
- 给新建 / 重命名 / 删除弹窗再统一一版视觉语言
- 增加更多树操作，例如复制路径、复制文件内容
- 给编辑器增加“已保存 / 未保存”状态提示
- 进一步优化空工作区的首屏引导，但目前已按要求隐藏空态文案
- 把当前的轻量阶段化编排继续拆成更清晰的 Planner / Executor / Reviewer / Summarizer 模块
