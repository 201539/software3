# 项目模板与骨架生成系统说明

## 概述

项目模板与骨架生成系统是 AI Coding Agent Web MVP 的核心功能之一，允许 Agent 从用户需求直接生成可运行的项目模板。系统包含 5 种常用项目模板，支持前端、后端、全栈应用的快速启动。

## 已支持的模板

### 1. Vite + React + TypeScript (`vite-react-ts`)

**分类**: 前端项目  
**描述**: 现代化 React 项目，包含 Vite 热更新、TypeScript 支持

**包含文件**:

- `index.html` - HTML 入口
- `vite.config.ts` - Vite 配置
- `tsconfig.json` - TypeScript 配置
- `src/main.tsx` - React 应用入口
- `src/App.tsx` - 主应用组件
- `src/index.css` - 全局样式
- `src/App.css` - 组件样式

```bash

npm run dev
```

## 使用方式

### 方式 1: 通过前端界面

1. 打开应用，在工作区文件树面板的"新建"按钮中选择"📦 生成项目模板"
2. 在模板选择对话框中选择想要的项目模板
3. 输入项目名称（默认为 `my-project`）
4. 点击"创建"，系统将自动生成项目骨架
5. 生成完成后，可以开始编辑文件或继续通过对话框让 Agent 修改项目

### 方式 2: 通过 API 调用

#### 获取所有模板

```bash
curl http://localhost:3000/api/templates
```

响应示例:

```json
{
  "templates": [
    {
      "id": "vite-react-ts",
      "name": "Vite + React + TypeScript",
      "description": "使用 Vite 构建的现代化 React 项目",
      "category": "frontend",
      "language": "typescript",
      "framework": "React",
      "version": "18.0"
    },
    ...
  ]
}
```

#### 按类别获取模板

```bash
curl http://localhost:3000/api/templates/category/frontend
```

#### 获取单个模板详情

```bash
curl http://localhost:3000/api/templates/vite-react-ts
```

#### 生成项目骨架

```bash
curl -X POST http://localhost:3000/api/scaffold/generate \
  -H "Content-Type: application/json" \
  -d '{
    "projectName": "my-awesome-app",
    "templateId": "vite-react-ts",
    "author": "Your Name",
    "description": "My awesome project"
  }'
```

该端点返回 SSE 流式响应，实时显示项目生成进度。

## 模板文件结构

每个模板由以下部分组成:

```typescript
{
  id: string;                           // 模板唯一标识
  name: string;                         // 模板显示名称
  description: string;                  // 模板描述
  category: 'frontend' | 'backend' |   // 模板类别
           'fullstack' | 'api' | 'cli';
  language: string;                     // 主要编程语言
  framework?: string;                   // 使用的框架
  version?: string;                     // 版本号
  files: Array<{                        // 模板包含的文件
    path: string;                       // 文件路径（支持 {{projectName}} 占位符）
    content: string;                    // 文件内容（支持 {{projectName}} 等占位符）
  }>;
  options?: {                           // 可选的参数选项
    [key: string]: Array<{
      name: string;
      value: string;
      type: string;
    }>;
  };
}
```

## 占位符支持

模板支持以下占位符，自动替换为用户输入的值:

- `{{projectName}}` - 项目名称
- `{{author}}` - 作者名称（如提供）
- `{{description}}` - 项目描述（如提供）
- `{{customParam}}` - 自定义参数（如定义）

## 扩展模板系统

### 添加新模板

1. 在 `packages/template-generator/templates.ts` 中添加新的模板定义:

```typescript
export const templates: Record<string, TemplateDefinition> = {
  // ... 现有模板 ...

  "my-template": {
    id: "my-template",
    name: "My Custom Template",
    description: "Description of my template",
    category: "frontend",
    language: "javascript",
    framework: "Vue",
    version: "3.0",
    files: [
      {
        path: "package.json",
        content: `{
  "name": "{{projectName}}",
  "version": "1.0.0"
}`,
      },
      // 添加更多文件...
    ],
  },
};
```

2. 重启应用，新模板会自动出现在模板列表中

### 自定义模板参数

模板可以支持自定义参数，用户在生成时可以选择:

```typescript
options?: {
  styling: [
    { name: 'Tailwind CSS', value: 'tailwind', type: 'string' },
    { name: 'CSS Modules', value: 'modules', type: 'string' },
  ],
  testing: [
    { name: 'Jest', value: 'jest', type: 'string' },
    { name: 'Vitest', value: 'vitest', type: 'string' },
  ],
}
```

## API 参考

### GET /api/templates

获取所有可用的模板。

**响应**:

```json
{
  "templates": [
    {
      "id": "template-id",
      "name": "Template Name",
      "description": "Template description",
      "category": "frontend",
      "language": "javascript",
      "framework": "React",
      "version": "18.0"
    }
  ]
}
```

### GET /api/templates/category/:category

按类别获取模板。

**参数**:

- `category` (string) - 模板类别 (frontend, backend, fullstack, api, cli)

**响应**:

```json
{
  "category": "frontend",
  "templates": [...]
}
```

### GET /api/templates/:id

获取单个模板的详细信息。

**参数**:

- `id` (string) - 模板 ID

**响应**:

```json
{
  "id": "vite-react-ts",
  "name": "Vite + React + TypeScript",
  "description": "...",
  "category": "frontend",
  "language": "typescript",
  "framework": "React",
  "version": "18.0",
  "fileCount": 11,
  "options": {}
}
```

### POST /api/scaffold/generate

生成项目骨架。返回 SSE 流式响应。

**请求体**:

```json
{
  "projectName": "my-project",
  "templateId": "vite-react-ts",
  "author": "Your Name",
  "description": "My project description"
}
```

**流式事件**:

```
data: {"type": "tool", "tool": "write_file", "summary": "创建文件: package.json"}
data: {"type": "tool", "tool": "write_file", "summary": "创建文件: src/App.tsx"}
...
data: {"type": "result", "result": {...}}
data: [DONE]
```

## 错误处理

系统会捕获以下常见错误:

1. **模板不存在** - 返回 404 错误
2. **项目名称无效** - 提示用户重新输入
3. **文件写入失败** - 显示详细错误信息
4. **模板生成失败** - 显示具体失败原因

## 性能考虑

- 模板是静态定义，不需要数据库查询
- 文件生成采用流式方式，逐个文件写入工作区
- 前端实时更新进度，提高用户体验
- 支持大型项目（100+ 文件）的快速生成

## 未来扩展方向

1. **动态模板加载** - 从远程源或数据库加载模板
2. **自定义参数支持** - 用户选择不同的配置选项
3. **多语言模板** - 支持不同编程语言的版本选择
4. **模板预览** - 在生成前预览项目结构
5. **模板验证** - 生成后自动验证项目的正确性
6. **依赖优化** - 自动选择最适合的依赖版本
7. **Git 初始化** - 自动初始化 Git 仓库和首次提交

## 故障排查

### 问题：模板生成失败

**解决方案**:

1. 检查工作区是否为空或有写权限
2. 查看浏览器控制台的错误信息
3. 检查项目名称是否有效（不含特殊字符）
4. 重启应用并重试

### 问题：文件没有正确生成

**解决方案**:

1. 刷新工作区文件树
2. 检查项目名称的替换是否正确
3. 查看生成日志中是否有错误消息
