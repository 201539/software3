# 快速开始：使用模板生成项目骨架

## 5 分钟快速入门

### 第 1 步：启动应用

```bash
# 进入项目目录
cd aicoding_ts

# 启动应用（包含 Web 服务和 Runtime）
npm run dev
```

应用会在 http://localhost:3000 启动。

### 第 2 步：打开应用并生成项目

1. 在浏览器打开应用
2. 找到右侧"工作区文件树"面板的"新建"按钮
3. 点击"新建" → 选择"📦 生成项目模板"
4. 在模板选择对话框中选择你想要的项目模板：
   - **前端项目**: Vite + React, Create React App
   - **后端项目**: Express API, Spring Boot
   - **全栈项目**: Next.js 应用
5. 输入项目名称（例如：`my-awesome-app`）
6. 点击"创建"

### 第 3 步：等待项目生成

系统会流式生成项目文件，你可以在对话区看到实时进度：

```
正在生成项目…
✓ 已生成 1 个文件…
✓ 已生成 2 个文件…
...
✅ 项目骨架生成完成！
```

### 第 4 步：开始开发

生成完成后，你可以：

- 在文件树中浏览生成的项目结构
- 点击文件查看和编辑代码
- 继续输入 Agent 命令来修改或添加功能

## 常见任务

### 任务 1：快速生成一个 React 应用

1. 点击"新建" → "📦 生成项目模板"
2. 选择 "Vite + React + TypeScript"
3. 输入项目名称：`my-react-app`
4. 等待生成完成
5. 打开 `src/App.tsx` 开始编辑

### 任务 2：创建一个 Express API 服务

1. 点击"新建" → "📦 生成项目模板"
2. 选择 "Express API 服务"
3. 输入项目名称：`my-api-server`
4. 打开 `src/routes/index.ts` 添加自己的路由

### 任务 3：启动 Next.js 全栈应用

1. 点击"新建" → "📦 生成项目模板"
2. 选择 "Next.js 应用"
3. 输入项目名称：`my-fullstack-app`
4. 生成后编辑 `app/page.tsx` 自定义首页

## API 调用示例

### 使用 curl 生成项目

```bash
# 生成 Vite + React 项目
curl -X POST http://localhost:3000/api/scaffold/generate \
  -H "Content-Type: application/json" \
  -d '{
    "projectName": "my-project",
    "templateId": "vite-react-ts"
  }' \
  --no-buffer
```

### 获取所有可用模板

```bash
curl http://localhost:3000/api/templates | jq
```

### 按类别获取模板

```bash
# 获取所有前端项目模板
curl http://localhost:3000/api/templates/category/frontend | jq

# 获取所有后端项目模板
curl http://localhost:3000/api/templates/category/backend | jq
```

## 故障排查

### 问题：生成按钮找不到

**解决方案**：检查左侧"工作区文件树"面板，找到顶部的"新建"按钮，点击后会出现下拉菜单。

### 问题：模板选择对话框无法打开

**解决方案**：

1. 刷新页面
2. 检查浏览器控制台是否有错误
3. 确保应用服务器正在运行

### 问题：文件没有生成到工作区

**解决方案**：

1. 检查工作区目录权限
2. 查看浏览器控制台的错误信息
3. 尝试使用不同的项目名称
