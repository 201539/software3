# 跨对话知识共享优化总结

## 优化目标

根据迭代三设计文档 3.2“跨对话知识共享”的要求，本次在 `Context-Engineering` 分支上补齐项目级知识库能力，让 Agent 可以在不同会话之间共享和复用项目经验。

## 已实现能力

### 1. 项目级知识库存储

修改文件：`packages/session-store/index.ts`

- 使用 `workspaces/<projectId>/project-memory.md` 作为项目级知识库。
- 新增默认 Markdown 模板，覆盖编码规范、技术决策、常见问题、用户偏好、Agent 任务经验。
- 新增 `getProjectMemory()`，返回内容、文件路径、是否已存在和默认模板。
- 新增 `writeProjectMemory()`，支持用户完整编辑并保存 Markdown。
- 新增 `appendProjectMemory()`，支持将任务经验追加到指定知识分区。

### 2. 项目知识 API

修改文件：`apps/runtime/server.ts`

新增接口：

- `GET /api/project-memory`：读取项目知识；文件不存在时返回默认模板。
- `PUT /api/project-memory`：覆盖保存项目知识。
- `POST /api/project-memory/append`：追加一条经验到指定分区，默认写入 `Agent 任务经验`。

这些接口让项目知识可以被 UI、Agent 流程和未来自动提取机制复用。

### 3. Context Builder 检索集成

修改文件：`packages/context-builder/index.ts`

上一轮上下文工程优化已完成以下集成，本次继续沿用：

- `buildForPrompt(prompt, selectedFile, { projectMemory })` 接收项目知识。
- 小于约 2KB 的知识全量注入。
- 较大的 `project-memory.md` 会按 Markdown 标题拆分段落。
- 使用 prompt token 匹配相关段落，只把命中的项目知识注入上下文。
- 检索结果进入 `projectMemorySummary`，由 Agent system prompt 注入。

### 4. Agent 任务经验沉淀建议

修改文件：`packages/agent-core/index.ts`

- 任务完成后，根据用户需求、已修改文件和使用工具生成“项目知识建议”。
- 建议文本被追加到任务摘要中，提醒用户哪些经验适合保存为跨会话共享知识。

### 5. Web UI 项目知识入口

修改文件：

- `apps/web/index.html`
- `apps/web/app.ts`
- `apps/web/styles.css`

新增“项目知识”面板：

- 支持查看和编辑 Markdown 格式项目记忆。
- 支持保存完整项目知识。
- 支持重载项目知识。
- 支持一键将最近任务摘要保存到 `Agent 任务经验`。
- 保存后的内容会立即刷新到编辑器中，后续会话执行任务时可被检索注入。
