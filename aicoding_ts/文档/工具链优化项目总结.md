## 优化目标

根据迭代三设计文档 §4「方向四：工具链重构与扩展」的要求，本次负责实现 Agent 工具链的安全增强、新工具扩展、patch 容错、调用可观测性与失败降级，使 `run_command` 具备 Cursor Composer（CC）风格的用户确认与白名单机制，降低 Agent 误执行危险命令的风险。

## 已实现能力

### 1. `run_command` 安全执行与 CC 风格确认

**新增文件**：

- `packages/tool-gateway/command-safety.ts` — 命令校验与风险分级
- `packages/tool-gateway/command-whitelist-store.ts` — 白名单持久化
- `packages/tool-gateway/run-command.ts` — 安全执行与确认 Hook

**核心逻辑**：

- **风险分级**：`low` / `medium` / `high`，依据命令首 token、危险子串（如 `rm -rf`、`git push --force`）判定
- **危险模式拦截**：禁止 `;` `&&` `|` 等 shell 元字符；禁止通过 `bash`/`cmd`/`powershell` 包装执行
- **白名单匹配**：支持 `exact`（完全匹配）、`prefix`（前缀）、`command`（首命令）三种规则；内置 `npm test`、`npm run *`、`git status` 等默认规则
- **确认流**：非白名单命令暂停 Agent 执行，经 SSE 推送 `command_confirm_request`；用户可选 **拒绝** / **允许一次** / **加入白名单并执行**
- **安全执行**：使用 `execFile` 无 shell 模式，按风险等级设置超时；白名单命中则直接执行

**持久化路径**：`workspaces/<projectId>/command-whitelist.json`

**修改文件**：`packages/tool-gateway/index.ts`、`packages/agent-core/executor.ts`、`packages/shared/types.ts`

### 2. 命令白名单与确认 API

**修改文件**：`apps/runtime/server.ts`

新增接口：

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/agent/command-confirm` | 提交命令确认结果（拒绝 / 一次 / 加白名单） |
| GET | `/api/command-whitelist` | 读取当前项目白名单 |
| POST | `/api/command-whitelist` | 手动添加白名单规则 |
| DELETE | `/api/command-whitelist/:id` | 删除指定规则 |

### 3. 新增 Agent 工具

#### `read_lints`

**文件**：`packages/tool-gateway/read-lints.ts`

- 运行 `npx tsc --noEmit` 进行 TypeScript 类型检查
- 补充启发式语法检查（未闭合括号、常见拼写等）
- 只读工具，无需用户确认，供 Agent 在修改代码后自检

#### `diff_file`

**文件**：`packages/tool-gateway/diff-file.ts`

- 对比当前磁盘文件与工作区版本快照的行级 diff
- 返回 `equal` / `add` / `remove` 类型 hunk 及增删统计
- 便于 Agent 在 patch 失败或 review 阶段查看变更

**注册位置**：`packages/agent-core/executor.ts`、`packages/agent-core/tool-definitions.ts`

### 4. `patch_file` 容错增强

**新增文件**：`packages/tool-gateway/patch-matcher.ts`

**修改文件**：`packages/workspace-manager/index.ts`

- 换行符归一化（`\r\n` → `\n`），解决 Windows 下 patch 匹配失败
- **Fuzzy 块匹配**：滑动窗口允许少量行偏差（默认最多 2 行不匹配）
- **行号锚点**：支持 `@@ line 42` 或 `# line 42` 格式，从指定行附近定位替换块
- 匹配失败时返回更明确的错误提示，引导 Agent 使用 `read_file` 核对原文

### 5. 工具调用日志与失败降级

**新增文件**：

- `packages/tool-gateway/tool-call-log.ts` — 内存环形缓冲，记录每次调用的参数摘要、结果状态、耗时
- `packages/tool-gateway/tool-fallback.ts` — 工具失败时的结构化 fallback 建议

**能力**：

- `GET /api/tools/:name/logs` 查询指定工具最近调用记录
- `enrichToolResult()` 在工具返回错误时附加 `fallback.message` 与 `suggestTools`，引导 Agent 换用其他工具
- 针对 `patch_file`、`run_command`、`read_lints` 等工具定制降级文案

### 6. 工具描述优化与禁用过滤

**新增文件**：`packages/agent-core/tool-definitions.ts`

- 将工具 schema 从 `executor.ts` 抽离，为每个工具补充使用示例与注意事项
- 已禁用的工具不再暴露给 LLM，避免 Agent 调用被禁用的能力

### 7. Web UI

**修改文件**：`apps/web/index.html`、`apps/web/app.ts`、`apps/web/styles.css`

- **命令确认弹窗**：展示命令、cwd、风险等级、拦截原因；三个操作按钮
- **命令白名单面板**：侧栏查看 / 手动添加 / 删除规则
- **工具管理增强**：工具测试对话框、调用日志列表、`diff_file` 结果在对话内格式化展示
