# AI Coding Agent Web 软件产品目标与 MVP 说明

## 1. 产品定位

这是一款面向全栈工程师的 AI Coding Agent Web 软件。

它的目标不是做成一个功能臃肿的“全能 IDE”，而是做成一个更轻量、更顺手、更适合中文用户的 AI 编码工作台。

本产品的核心不是强制把系统拆成传统前后端应用，也不是强制依赖 API 驱动一切，而是围绕“Agent 能实际完成任务”来设计：

- 需要网页交互时，就提供 Chat、编辑器和工作区
- 需要数据库时，再引入数据库
- 不需要数据库时，就先不强行上
- 需要命令行能力时，Agent 可以直接调用 CLI 来完成修改、检查、生成和验证
- 整体体验可以更接近 Claude Code 这类“以命令行执行任务”的工作方式，只是在 Web 里提供更友好的可视化和交互入口

### 产品形态

- Chat
- 代码编辑器
- 工作区文件列表展示
- Agent 命令执行面板 / 终端能力

### 核心特点

- 比 Cursor 更轻量
- 更适合中文用户的使用习惯
- 聚焦于“生成项目骨架 + 继续对话修改”的主流程
- 让用户快速从需求走到可编辑的项目代码
- 支持以命令行方式完成修改、检查和验证，不强制依赖 API 通信模式

---

## 2. 目标用户

### 主要用户

- 全栈工程师

### 用户特点

- 能读代码，也能改代码
- 需要快速搭项目和改业务
- 希望 AI 能提高开发效率，而不是增加理解成本
- 更关心“能不能直接开始干活”，而不是复杂配置
- 对命令行和代码仓库工作流比较熟悉，能接受 Agent 直接动文件、跑命令、做验证

---

## 3. 产品要解决的问题

本产品主要解决以下问题：

1. 让用户快速生成可运行的项目骨架
2. 让用户在生成后可以继续对话修改代码
3. 让用户能清楚看到工作区文件结构
4. 让用户能直接在网页里编辑代码
5. 让用户不用面对过重、过复杂、过多干扰的界面
6. 让中文用户能更自然地表达需求并得到结果
7. 让 Agent 能像 Claude Code 一样直接通过命令行完成部分开发任务
8. 不强制要求所有能力都通过 API 或前后端分层来实现，而是按实际需求选择最合适的执行方式

---

## 4. 产品范围

### 4.1 核心范围

- 用户输入需求
- 系统生成项目骨架
- 展示项目文件列表
- 展示代码编辑器
- 支持继续对话修改项目
- 支持查看和编辑工作区文件
- 支持 Agent 调用命令行执行任务
- 支持 Agent 根据需要修改文档、代码和配置文件

### 4.2 架构原则

- 不强制前后端固定模式
- 不强制所有能力都必须通过 API 通信
- 数据库不是前置必需项，只有当功能需要时再引入
- 优先使用最直接、最适合当前任务的执行方式
- Agent 需要能够在受控环境中执行命令、读取输出、修改文件并验证结果

### 4.3 暂不考虑的内容

- 赚钱模式设计
- 复杂协作
- 多人实时编辑
- 插件市场
- 复杂权限体系
- 企业组织管理
- 过重的 IDE 级功能

---

## 5. MVP 定义

### MVP 的唯一主流程

1. 用户输入需求
2. 系统生成项目骨架
3. 用户查看工作区文件列表
4. 用户在代码编辑器中查看和修改代码
5. 用户继续通过对话提出修改需求
6. 系统根据对话更新项目内容
7. 必要时，Agent 通过命令行执行生成、修改、检查和验证

### MVP 成功标准

如果一个新用户能在最短时间内完成以下动作，就说明 MVP 有价值：

- 输入一句需求
- 看到生成的项目结构
- 打开某个文件
- 继续让 AI 修改代码
- 必要时看到 Agent 通过命令行完成任务并输出结果
- 最终得到一个可继续开发的项目

## 6. 产品价值主张

### 对用户的价值

- 更快启动项目
- 更快修改项目
- 更少认知负担
- 更适合中文交流
- 更轻量，不打扰开发节奏
- 既能通过 Web 交互，也能借助命令行式执行方式完成更直接的任务

### 对产品的价值

- 聚焦一个清晰场景
- 容易形成稳定主流程
- 便于快速迭代
- 便于后续扩展而不失焦
- 架构上保持灵活，不被固定前后端/API 形式限制

---

## 7. 与 Cursor 的差异化

本产品不是要完全替代 Cursor，而是要在体验上做出更轻量的版本。

### 主要差异点

- 更轻量的界面
- 更少的复杂功能
- 更直接的中文交互
- 更聚焦“生成 + 修改”的主路径
- 更适合只想快速干活的全栈工程师
- 同时保留 Agent 命令行式执行能力，让系统在必要时像 Claude Code 一样工作

---

## 8. 使用 Cursor 辅助开发的建议

在开发这个产品时，可以把 Cursor 作为编码加速器，但产品方向和流程应该由你自己掌控。

### 推荐做法

- 先确定产品主流程，再让 Cursor 写代码
- 先拆模块，再逐个实现
- 每次只让 Cursor 处理一个小任务
- 让 Cursor 生成样板代码、表单、状态管理和重复逻辑
- 需要执行命令、验证结果、改文档或调试时，优先设计 Agent 的 CLI 能力，而不是强行把所有动作都绕成 API
- 核心架构、产品决策和体验设计由你主导

---

## 9. 这份产品的最简目标描述

这是一个面向全栈工程师的轻量级 AI Coding Agent Web 软件，采用 Chat + 代码编辑器 + 工作区文件列表的形态，重点解决项目骨架生成和后续对话修改问题，同时支持 Agent 使用命令行直接完成修改、检查和验证。产品架构不强制依赖前后端分层或固定 API 模式，数据库按需使用，目标是提供比 Cursor 更轻、更适合中文用户的开发体验。
