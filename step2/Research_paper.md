# Research Paper Assistant Agent 应用方案（迭代二）

## 1. 方案概述

本方案设计一个面向学术研究场景的自治 Agent：**Research Paper Assistant**。  
该 Agent 面向“从选题到论文初稿”的科研流程，提供任务规划、文献检索、证据整理、实验方案生成、写作辅助与自我修正能力。

与传统“问答式助手”不同，本应用强调 Agent 在开放任务中的自主决策：
- 根据研究目标动态规划执行路径；
- 调用外部学术工具获取与验证信息；
- 在证据不足或冲突时自动反思与重规划；
- 输出可追溯的研究过程与结果。

---

## 2. 应用定位与使用场景

### 2.1 目标用户
- 本科/研究生课程项目团队
- 科研入门阶段学生
- 需要高效完成文献综述与实验设计的研究者

### 2.2 典型使用场景
1. 给定研究主题，生成可执行的选题路线与时间计划；
2. 自动检索并筛选高相关论文，输出结构化阅读清单；
3. 对多篇论文进行方法对比与研究空白识别；
4. 根据目标生成实验设计建议（变量、指标、基线、数据集）；
5. 生成论文初稿大纲与段落草稿，并提供引用建议。

---

## 3. 功能需求（Agent 应用）

### 3.1 核心功能

1. **研究目标理解与任务分解**
   - 输入：研究主题、课程要求、时间约束、已有基础；
   - 输出：研究子任务列表（检索、阅读、对比、实验、写作）与里程碑。

2. **文献检索与筛选**
   - 多源检索候选文献并去重；
   - 按相关性、时效性、引用影响力进行排序；
   - 输出“必读/可选/补充”三级阅读列表。

3. **文献分析与知识整理**
   - 自动抽取研究问题、方法、数据集、指标、结论、局限；
   - 生成论文对比表与研究脉络图；
   - 识别争议点与潜在研究空白。

4. **实验设计与评估建议**
   - 生成实验方案（变量、对照组、评价指标、消融实验建议）；
   - 估算实验复杂度与风险；
   - 提供结果分析模板。

5. **写作支持与交付输出**
   - 生成论文大纲与章节草稿；
   - 输出引用建议与参考文献清单；
   - 生成周报/进展报告/下一步计划。

### 3.2 外部工具设计（至少 2 个）

> 满足课程要求：具备与外部世界交互能力，且工具与主题强相关。

- **Tool A：学术检索工具（Scholar Search Adapter）**
  - 能力：关键词检索、过滤年份/领域、返回论文元信息与链接；
  - 用途：构建候选文献池。

- **Tool B：文献解析工具（Paper Parser）**
  - 能力：解析 PDF/摘要，抽取方法、数据集、指标、结论、局限；
  - 用途：生成结构化文献卡片与对比分析表。

- （可选）**Tool C：引用与格式工具（Citation Formatter）**
  - 能力：BibTeX/APA/GB-T 7714 格式转换与去重；
  - 用途：支持论文写作阶段的规范化输出。

- （可选）**Tool D：实验记录工具（Experiment Tracker）**
  - 能力：记录实验配置、结果、版本；
  - 用途：支撑复现与评估。

### 3.3 自我修正机制

Agent 在以下情况下进行自动修正：
- 检索结果过少或偏题：自动改写关键词并扩展检索策略；
- 文献结论冲突：提高证据等级要求并补充高质量来源；
- 实验方案不可执行：根据资源约束重新规划轻量实验；
- 写作内容证据不足：回退到检索/阅读阶段补齐依据。

机制设计：
- 错误分类（检索错误/证据不足/逻辑冲突/格式异常）；
- 重试与替代策略；
- 反思日志（失败原因 -> 修正动作 -> 结果）。

### 3.4 状态可视化

建议提供 CLI 或 Web 可视化：
- 状态流：`Goal -> Plan -> Retrieve -> Analyze -> Synthesize -> Reflect -> Deliver`
- 可视化内容：
  - 当前研究阶段与进度；
  - 文献池统计（数量、年份分布、来源分布）；
  - 决策节点（为何选择某些论文/方法）；
  - 失败与修正路径。

## 4. 推理框架设计

## 4.1 组合式推理框架（推荐）

为体现 Agent 的自治与加分项，采用多框架协同：

1. **Plan-and-Execute（主干框架）**
   - 先生成研究计划（检索、阅读、实验、写作），再逐步执行；
   - 适合长期目标与多阶段任务。

2. **ReAct（工具交互执行）**
   - 在检索、解析、比对时进行“思考-行动-观察”循环；
   - 适合需要高频工具调用的阶段。

3. **Reflexion（冲突与失败反思）**
   - 在证据冲突、实验受限或输出质量不足时触发；
   - 强化自我修正和策略更新能力。

## 4.2 框架路由策略
- 简单问题（如单文献摘要）：轻量 ReAct；
- 完整研究任务：Plan-and-Execute + ReAct；
- 连续失败或冲突任务：触发 Reflexion 并重规划。

---

## 5. 系统架构设计

## 5.1 总体架构

采用前后端分离架构：

1. **前端层（Web）**
   - 输入研究目标与约束；
   - 查看计划、文献对比、实验建议与写作输出；
   - 可视化状态流转与执行日志。

2. **Agent Orchestrator（后端编排层）**
   - 负责任务分解、上下文管理、推理路由与异常恢复；
   - 控制整体执行流程与阶段切换。

3. **Reasoning Engine（推理层）**
   - 封装 ReAct / Plan-and-Execute / Reflexion；
   - 提供统一调用接口与可替换策略。

4. **Tool Adapter（工具层）**
   - 对接 Scholar Search、Paper Parser、Citation Formatter 等外部能力；
   - 统一 I/O 协议，支持后续扩展。

5. **Data & Trace（数据层）**
   - 存储任务、文献卡片、执行日志、错误记录、版本快照；
   - 支撑评估平台的过程指标与对比分析。

## 5.2 关键执行流程
1. 用户输入研究主题、时间范围与目标产物；
2. Agent 生成阶段计划与优先级；
3. 调用检索工具构建候选文献池；
4. 调用解析工具抽取结构化信息并生成对比；
5. 输出研究空白与实验方案；
6. 生成论文大纲与草稿；
7. 如发现证据不足或冲突，触发反思并重规划；
8. 输出最终交付与过程报告。

---

## 6. 非功能需求

1. **安全与合规**
   - 禁止硬编码 API Key；
   - 使用环境变量或配置系统管理密钥；
   - 日志脱敏，避免泄露隐私和受限资料。

2. **可维护性**
   - 推理层/工具层/数据层解耦；
   - 统一错误码与日志协议。

3. **可扩展性**
   - 可新增学术数据库与领域专属工具；
   - 可新增评估指标与策略。

4. **可观测性**
   - 提供任务级 Trace ID；
   - 支持回放执行轨迹与关键决策复盘。

## 7. 迭代二 MVP 交付范围

### 7.1 必做交付
- 可运行的 Research Paper Assistant Web 应用；
- 至少 2 个主题相关外部工具（Scholar Search + Paper Parser）；
- 至少 2 种推理框架协同（如 Plan-and-Execute + ReAct）；
- 自我修正机制（重试、替代策略、反思重规划）；
- 执行过程可视化；
- 演示视频覆盖完整闭环。

### 7.2 可选增强
- 引入 Reflexion 增强复杂场景收敛；
- 增加实验记录工具对接；
- 增加多方案自动比较与推荐。

---

## 8. 与任务二评估平台对接

Research Paper Assistant 需要输出结构化评估数据：
- 输入任务与上下文；
- 检索与筛选过程（命中率、覆盖率、重复率）；
- 文献分析与冲突修正次数；
- 工具调用成功率、耗时、失败类型；
- 最终产出质量（完整性、可追溯性、可执行性）。

这些数据可支撑：
- 显式指标：耗时、成功率、检索有效率、工具正确率；
- 模糊指标：推理质量、结论可信度、写作可读性；
- 过程评估：反思机制是否有效、修正是否降低错误率。

---

## 9. 风险与应对

1. **检索噪声高、文献质量不稳定**
   - 应对：多信号排序（相关性+年份+引用影响力）与人工确认入口。

2. **长上下文导致推理漂移**
   - 应对：分阶段摘要、关键证据缓存、上下文窗口管理。

3. **证据冲突导致输出不一致**
   - 应对：冲突检测规则 + 高质量来源优先 + 反思重规划。

4. **演示链路复杂易失败**
   - 应对：固定主题、固定文献集、标准演示脚本。

---

## 10. 总结

Research Paper Assistant 面向科研任务的开放性与不确定性，能够体现迭代二任务一要求的核心能力：
- 自治规划与动态执行；
- 多外部工具交互；
- 异常处理与自我修正；
- 状态流转可视化与过程可追溯；
- 与任务二评估平台自然联动。
