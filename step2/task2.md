# 迭代二任务二分析（Agent 评估平台）

## 1. 任务目标（你们要做什么）
搭建一个**Agent 应用评估平台**，用于配置、执行、分析和对比 Agent 的评测任务。  
本质上要解决的问题是：如何系统化地判断 Agent“好不好、稳不稳、快不快、安全不安全”。

课程给出的硬约束：
- 前后端分离 Web 项目；
- 评估指标可直接使用 Ragas，也可扩展自定义指标；
- 需要支持任务管理、执行、分析、可视化和对比。

---

## 2. 硬性需求拆解（逐条对应课程要求）

### 2.1 评测任务管理能力
需要有一个“评测任务中心”，支持完整生命周期管理。

至少应支持：
- 创建评测任务；
- 配置评测任务（Agent 版本、数据集、评测方法、参数）；
- 任务的增删改查（CRUD）；
- 任务状态管理与展示（待执行/运行中/成功/失败）。

建议数据模型（最小）：
- `evaluation_task`：基本信息、配置、状态、创建人、时间；
- `evaluation_run`：每次运行记录、版本快照、结果摘要；
- `dataset`：数据集元信息、样本数量、版本。

### 2.2 评测执行与方法能力
课程强调三种“划分视角”，你们平台要能覆盖。

#### A. 按评估模式
- **面向结果**：只看输入输出质量；
- **面向过程**：看中间过程（规划质量、工具使用、重试行为等）。

#### B. 按评估方式
- **显式指标**（可计算）：
  - Token 消耗；
  - 工具调用成功率/正确率；
  - 任务成功率；
  - 平均响应时间、P95 延迟。
- **模糊指标**（LLM-as-a-Judge）：
  - 推理质量；
  - 输出准确性；
  - 幻觉程度；
  - 交互体验。

#### C. 按评估维度
- **效果维度**：是否有效、准确、完整；
- **安全维度**：是否存在有害或违规输出；
- **性能维度**：是否流畅、等待是否可接受。

注意：这三种划分会重叠，不需要做成互斥分类树，而应做成可组合配置。

### 2.3 扩展能力
平台不能“写死”。至少要支持：
- 自定义指标插件化接入；
- 组合评估策略（例如加权得分、多指标门槛）。

建议最小机制：
- 指标注册表（name + 输入 schema + 输出 schema）；
- 策略配置（权重、阈值、聚合方法）；
- 指标执行器统一接口。

### 2.4 评测分析与展示能力
必须“看得懂结果”。

至少应支持：
- 单次任务可视化（总分、各指标、样本分布、失败案例）；
- 多任务对比（不同 Agent 版本、不同数据集、不同策略）；
- 趋势视图（版本迭代效果变化）。

推荐图表：
- 雷达图（效果/安全/性能）；
- 柱状图（版本对比）；
- 折线图（时间趋势）；
- 表格（样本级明细与可追溯日志）。

## 3. 推荐实现方案（MVP）

### 3.1 架构
- 前端：评测任务管理 + 结果分析看板；
- 后端：任务调度器 + 指标执行器 + 结果聚合器；
- 数据层：任务配置、运行日志、样本级结果、版本元数据。

### 3.2 执行链路
1. 用户创建评测任务并选择 Agent 版本/数据集/指标策略；
2. 调度器触发评测 run；
3. 平台调用 Agent，采集输入、输出与过程日志；
4. 指标模块计算显式指标 + LLM-as-a-Judge 指标；
5. 聚合结果，生成任务报告与可视化。

### 3.3 与任务一联动（非常关键）
为了评“过程”，任务一的 Agent 需输出结构化 trace：
- 计划步骤；
- 工具调用链；
- 重试与异常恢复；
- 最终产出和状态。

没有这类结构化日志，任务二很难做好“面向过程评估”。

## 4. 迭代二任务二交付清单（建议按此准备）
1. 前后端分离 Web 评估平台；
2. 评测任务管理（CRUD + 状态流转）；
3. 至少覆盖结果评估 + 过程评估各一类；
4. 至少包含显式指标 + 模糊指标各一类；
5. 自定义指标或组合策略能力（至少实现一个）；
6. 单次结果可视化 + 多次对比分析；
7. 演示视频展示“建任务 -> 执行 -> 分析 -> 对比”闭环。

---

## 5. 推荐里程碑（6 周节奏）
- **第 1-2 周**：完成评测任务管理与基础执行框架；
- **第 3 周**：接入显式指标与基础可视化；
- **第 4 周**：接入 LLM-as-a-Judge 与过程评估；
- **第 5 周**：完成对比分析、自定义指标/组合策略；
- **第 6 周**：联调、稳定性优化、录制演示视频。

---

## 6. 常见风险与规避
- **只做“单次打分脚本”而非平台** -> 必须有任务管理和状态；
- **只有总分没有样本细节** -> 增加样本级明细和失败案例；
- **只能评结果不能评过程** -> 尽早定义并采集 Agent trace；
- **评估不可复现** -> 固定数据集版本、评估配置与模型版本；
- **API Key 泄露风险** -> 全部改为环境变量/密钥管理。
